On desktop, a notification is only displayed for a short period of time. On Android, notifications are shown until the user interacts with it. To get the same behaviour on desktop and mobile you can set the "require-interaction" option to true, which means the user must click or dismiss the notification.

When you use the tag option, the default behavior of a new notification replacing an existing one is that there is no sound, vibration and the screen is kept asleep. With `renotify: true` a new notification will play a sound, vibrate and wake up the users device. This means replacing notifications have the same behavior as a completely new notification. Note: Therde is no visible affect on desktop, but on mobile, vibration and sound will be affected.


You must provide a subject that is either a mailto: or a URL.


You must provide host for notifications images.
